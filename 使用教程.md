## 前台页面介绍

### 🏠 首页（`/`）

打开网站看到的第一个页面，包含：
- **Hero 大标题区** —— 学校名称、标语、按钮（可选背景图）

- 学校详细介绍文字
- 学校照片

- 所有通知公告和新闻动态

## 后台管理（教师专用）

> 后台地址：**https://school-website.sea-hang.workers.dev/admin**

登录后可以看到两个管理区域：

### 一、➕ 添加活动

| 字段 | 说明 |
|------|------|
| 📰 新闻 / 📢 通知 | 选择类型 |
| 标题 | 必填，活动标题 |
| 日期 | 必填，如 `07-09` |
| 摘要 | 必填，简短描述 |

填写后点 **"添加活动"** 即可。

**已有活动列表**在下方，鼠标移上去会出现 🗑 删除按钮。

### 二、🖼️ 图片设置

可以更改网站四个位置的图片：

| 字段 | 显示在哪里 |
|------|-----------|
| `heroBgImage` | 首页顶部大背景（可留空，不显示背景图） |
| `aboutPreviewImage` | 首页"关于红旗实验学校"旁边的照片 |
| `aboutImage` | 学校简介页（`/about`）的照片 |
| `contactMapImage` | 联系方式页（`/contact`）的照片 |

## 如何更换图片

### 方法一：直接编辑配置文件（推荐 ⭐）

最简单的方式，直接改配置文件然后部署：

**步骤：**

1. 把图片放到 `public/uploads/activities/` 文件夹
2. 打开 `src/data/settings.json`，填入图片路径：

```json
{
  "heroBgImage": "/uploads/activities/背景图.jpg",
  "aboutPreviewImage": "/uploads/activities/学校照片.jpg",
  "aboutImage": "/uploads/activities/学校照片.jpg",
  "contactMapImage": "/uploads/activities/学校照片.jpg"
}
```

3. 运行部署命令：

```bash
npm run deploy
```

等 1-2 分钟，刷新网站就能看到新图片。

> 💡 **提示**：不需要的图片位置直接删掉那一行就行，比如不想要背景图就删掉 `heroBgImage` 那一行。一张图片可以用在多个位置，路径填一样的即可。

### 方法二：通过线上后台

1. 浏览器打开 **https://school-website.sea-hang.workers.dev/admin**
2. 教师登录（输入教师密码）
3. 在 🖼️ 图片设置区域填入图片路径
4. 点 **保存图片设置**（即时生效，无需部署）

> ⚠️ 注意：后台保存的设置会覆盖 `settings.json` 里的默认值。如果想让文件设置恢复生效，在后台把输入框清空后保存即可。

### 图片文件要求

- 放的位置：`public/uploads/activities/`
- 支持的格式：jpg、png、gif、webp
- 建议英文文件名（如 `school.jpg`）
- 建议图片大小在 500KB 以内

## 如何发布活动/新闻

1. 打开后台管理页面：https://school-website.sea-hang.workers.dev/admin
2. 教师登录（输入教师密码）
3. 在 ➕ 添加新活动区域：
   - 选择 📰 新闻 或 📢 通知
   - 填写标题、日期、摘要
   - 如果是通知，可以加上标签（如"重要"、"紧急"）
4. 点 **添加活动**
5. 发布后自动显示在首页和活动栏页面

**删除活动**：在已有活动列表里，鼠标移上去，点 🗑 按钮。

> 💡 活动数据存在云端，不需要运行 `npm run deploy`。

## 如何修改页面文字

打开 `src/data/` 文件夹，直接编辑对应的文件，然后运行 `npm run deploy`：

| 文件 | 控制什么 |
|------|---------|
| `site.ts` | 校名、导航、页脚、地址、电话、邮箱 |
| `home-hero.ts` | 首页大标题、标语、按钮文字 |
| `home.ts` | 首页"学校简介"区块的文字 |
| `about.ts` | 学校简介页的完整文字、办学理念 |
| `contact.ts` | 联系方式页的地址、电话、邮箱、办公时间 |

## 本地开发指南

### 环境要求

- Node.js 22+
- npm 10+

### 安装和运行

```bash
# 进入项目文件夹
cd school-website

# 安装依赖（第一次）
npm install

# 启动本地开发服务器
npm run dev

# 浏览器打开
# http://localhost:3000
```

### 项目结构

```
school-website/
├── src/
│   ├── app/              # 各页面
│   │   ├── page.tsx      # 首页
│   │   ├── about/        # 学校简介
│   │   ├── activities/   # 活动栏
│   │   ├── contact/      # 联系方式
│   │   ├── login/        # 登录页
│   │   ├── admin/        # 后台管理
│   │   └── api/          # API 接口
│   ├── components/       # 页面组件
│   │   ├── sections/     # 首页各区块
│   │   └── ui/           # 通用 UI 组件
│   ├── data/             # 📝 可编辑的配置文件
│   │   ├── site.ts       # 学校信息、导航
│   │   ├── home-hero.ts  # 首页大标题
│   │   ├── home.ts       # 首页其他内容
│   │   ├── about.ts      # 学校简介内容
│   │   ├── contact.ts    # 联系方式内容
│   │   ├── activities.json  # 活动数据（云端存储）
│   │   └── settings.json    # 图片设置（默认值）
│   └── lib/              # 工具函数
│       └── settings-store.ts  # 设置存储（本地文件 + 云端 KV）
├── public/uploads/activities/  # 🖼️ 放图片的地方
├── wrangler.jsonc        # Cloudflare 配置
└── package.json          # 项目脚本
```

### 常用命令

| 命令 | 作用 |
|------|------|
| `npm run dev` | 启动本地服务器 |
| `npm run build` | 构建（检查代码是否有错） |
| `npm run deploy` | 部署到线上（含 Git 推送） |
| `npm run preview` | 本地预览线上版本 |

## 常见问题

### Q：忘记教师密码了怎么办？

请联系学校管理员获取教师密码。

### Q：换了图片但网页上没变化？

1. 确认图片已放在 `public/uploads/activities/` 文件夹
2. 确认 `settings.json` 里路径写对了
3. 确认已运行 `npm run deploy`
4. 等 1-2 分钟，Cloudflare 需要时间更新
5. 如果之前通过后台保存过图片设置，后台设置会覆盖文件设置。去后台把图片输入框清空再保存，文件设置就会生效。

### Q：后台保存的图片设置和 settings.json 哪个优先？

后台保存的优先。后台设置存在云端 KV 数据库里，会覆盖 `settings.json` 的默认值。想让 `settings.json` 生效，去后台清空图片设置后保存即可。

### Q：npm run deploy 报错？

```bash
# 先构建检查错误
npm run build

# 如果构建成功，再部署
npm run deploy
```

### Q：怎么在另一台电脑上管理网站？

```bash
# 1. 下载项目
git clone https://github.com/Sea-hang/hongflag.git
cd hongflag

# 2. 安装依赖
npm install

# 3. 就可以开始用了
npm run dev
```

### Q：网站访问慢怎么办？

网站托管在 Cloudflare 全球网络，首次访问可能需要 1-2 秒加载，之后会很快。

### Q：图片太大怎么办？

建议图片在 500KB 以内。可以用电脑自带的预览工具缩小图片尺寸。
